Title: User Controller (Schema Kit)
Scope: schema-kit module
Applies to: Nuxt 4 + TRPC + SurrealDB

This document explains the **User controller** generated by Schema Kit.
It is intentionally explicit so an AI can follow it without confusion.

---

## 1) What this controller is

The **User controller** is a front-end helper that wraps the TRPC router for the `user` model.
It gives you simple methods like `create`, `update`, `delete`, `get`, `taxonomy`, `subtable`, `relation`, and `typesense` (when enabled).

---

## 2) Where the controller lives

Generated base controller:
- `module/src/runtime/controllers/generated/user.ts`

Wrapper that merges generated + custom overrides:
- `module/src/runtime/controllers/user.ts`

Custom overrides (project side):
- `schema/controllers/user.ts` (or `app/schema/controllers/user.ts`)

---

## 3) How overrides work

The wrapper calls:
- `createGeneratedUserController()`
- `extendUserController` (your custom overrides)

If a custom function has the **same name** as a generated function, the custom one **replaces** it.
If it’s a new function, it is added.

Example override:

```ts
import type { UserController } from '@schema/controllers/user'

export const extendUserController = (base: UserController) => ({
  async update(id, payload, options) {
    return base.update(id, payload, options)
  },
  async customThing(id) {
    return base.get(id, 'Admin')
  },
})
```

---

## 4) ID normalization (very important)

SurrealDB returns record ids as **objects** like:
```ts
{ tb: 'u', id: '123' }
```

TRPC endpoints expect the **sub-id**, which for this model is:
`string`

The controller accepts **either**:
- a record object
- a sub-id
- a full record object
- arrays/tuples of ids

It always normalizes to the sub-id before calling TRPC.
